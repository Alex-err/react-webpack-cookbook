Installing webpack

  • In your project folder: npm init
  • npm install webpack -g , Installs webpack globally
  • optional npm install webpack , Installs a project version of webpack

Creating a configuration

Create a file in your project root named: webpack.config.js.

webpack.config.js

var path = require('path');
var config = {
  entry: path.resolve(__dirname, "APP_ENTRY.js") // E.g. "app/main.js",
  output: {
    path: path.resolve(__dirname, "BUILD_DIR"), // E.g. "build"
    filename: "BUNDLE_NAME.js" // E.g. "bundle.js"
  }
};

module.exports = config;

Folder structure

  • /app
    • main.js
  • /build
    • bundle.js
  • package.json
  • webpack.config.js

You can of course define this structure yourself, but webpack will need a folder where your bundled files and chunks will run from.

Creating your first app

Let us create a simple application that says "Hello world". We start off by creating a Component.js file in our /app folder.

Component.js

export default function () {
  var element = document.createElement('h1');
  element.innerHTML = 'Hello world';
  return element;
};

And lets update our main.js file to render this component to the body.

main.js

import Component from './Component.js';
document.body.appendChild(Component());

Building the application

Run webpack in your terminal and your application will be built. A bundle.js file will appear in your /build folder.
