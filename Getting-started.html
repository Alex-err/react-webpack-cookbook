Installing webpack

  1. In your project folder: npm init
  2. npm install webpack -g (Installs webpack globally)
  3. optional: npm install webpack , (Installs a project version of webpack)

Folder structure

  • /app
    • main.js
  • /build
    • bundle.js (automatically created)
    • index.html
  • package.json
  • webpack.config.js

You can of course define this structure yourself, but Webpack will need an output folder. In this example build/.

Creating a configuration

Create a file in your project root named: webpack.config.js.

webpack.config.js

var path = require('path');
var config = {
  entry: path.resolve(__dirname, "APP_ENTRY.js") // E.g. "app/main.js",
  output: {
    path: path.resolve(__dirname, "BUILD_DIR"), // E.g. "build"
    filename: "BUNDLE_NAME.js" // E.g. "bundle.js"
  }
};

module.exports = config;

Running your first build

Let us create a simple build that says "Hello world" when run in the browser. We start off by creating a Component.js file in our /app folder.

app/Component.js

export default function () {
  var element = document.createElement('h1');
  element.innerHTML = 'Hello world';
  return element;
};

And lets update our main.js file to render this component to the body.

app/main.js

import Component from './Component.js';
document.body.appendChild(Component());

Now run webpack in your terminal and your application will be built. A bundle.js file will appear in your /build folder. Your index.html file in the build/ folder will need to load up the application.

build/index.html

<!DOCTYPE html>
<html>
  <head>
    <meta charset="UTF-8"/>
  </head>
  <body>
    <script src="bundle.js"></script>
  </body>
</html>

Running the application

Just double-click the index.html file or set up a web server pointing to the build/ folder.
